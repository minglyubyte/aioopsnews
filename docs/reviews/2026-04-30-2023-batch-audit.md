# 2023 Batch Audit

Source file reviewed: [2023_batch.csv](/Users/leo/Downloads/2023_batch.csv)

## Goal

Review each row for:

- whether it looks like a legitimate AI-related incident for the product
- whether it satisfies the new minimum-source rule
- whether it should be accepted, reviewed further, or rejected

## Legend

- `ACCEPT`: Legit AI-related incident and worth keeping after source expansion
- `REVIEW`: Plausible or likely legit, but source quality, specificity, framing, or date needs more work
- `REJECT`: Not strong enough, not really a discrete incident, wrong year, or misleading framing
- `Conf`: confidence in this editorial judgment, not confidence in the incident itself
- `Links`: current source count out of required 3

## Top Findings

- The file contains `100` incidents, not `79`.
- `0/100` rows meet the new `3 links minimum`.
- `42/100` rows have `2/3` links and need `+1`.
- `58/100` rows have `1/3` links and need `+2`.
- The current CSV schema only has `primary_source` and `secondary_source`, so it cannot satisfy the new rule without either:
  - a third source column, or
  - list-style source columns that allow multiple links per source type
